# note 記事フォーマット

## 基本ルール
- タイトル: 32文字以内、具体的な数字入り
- 冒頭: 「この記事を読むと何が得られるか」を3行以内で明示
- 見出し: h2/h3で構造化（h2は3-5個が目安）
- 改行: 3-4行ごとに空行を入れる（詰まった印象を避ける）
- 文字数: 2000-4000文字が読了率の最適ゾーン
- 最後: まとめ + CTA（ブログへの誘導）

## タイトルの付け方
- 数字を入れる（「3つの理由」「月5万円」「2週間で」）
- 具体的な結果・変化を示す
- 疑問形 or 断定形（「〜の方法」「〜した話」）
- 良い例: 「電脳せどりで月5万稼ぐまでにやった3つのこと」
- 良い例: 「店舗せどりで失敗した僕が、電脳に切り替えて変わった話」
- NG例: 「せどりについて」（抽象的すぎる）
- NG例: 「最強のせどり術を大公開！」（煽りすぎ）

## 構成テンプレート

```
# タイトル（32文字以内・数字入り）

[冒頭3行: この記事で得られること]

## 見出し1（背景・きっかけ）
体験ベースで語る。「僕が〜したとき」から始める。

## 見出し2（本題・具体的な内容）
数字や具体例を交えて。スクショや実績があれば言及。

## 見出し3（学び・気づき）
やってみてわかったこと。失敗も正直に。

## まとめ
3行以内で要点を振り返る。

---
もっと詳しい情報はブログで書いてます。
よければ覗いてみてください。
[ブログURL]
```

## SEOカニバリ対策
- ブログと同じキーワードで書かない
- 切り口を変える（ブログ=ノウハウ、note=体験談/ストーリー）
- noteは「人」にフォーカス、ブログは「情報」にフォーカス
- 例: ブログ「電脳せどり おすすめサイト5選」→ note「電脳せどりを始めて2ヶ月。正直な感想を書く」

## noteで読まれるコツ
- 冒頭100文字が勝負（ここで離脱される）
- 「自分の話」として書く（一般論は読まれない）
- 完璧な成功談より「泥臭いプロセス」
- 最後まで読んだ人への感謝を入れる
- マガジン登録を意識（シリーズ化できるテーマ）

## NGパターン
- 情報の羅列（「〇〇とは」「メリットは」→ それはブログでやれ）
- 冒頭が長い前置き
- CTA がゴリゴリの宣伝
- 他プラットフォームの使い回し感
