# ビジネスメール作成スキル

ビジネスメールの文面を作成するスキル。
**日付・曜日の正確性を最優先する。**

## 絶対ルール

### 1. 日付・曜日の検証（最重要）
- 日付が含まれる場合、**必ずカレンダー計算で曜日を検証してから出力する**
- 曜日の計算方法: ツェラーの公式またはシステム日付から逆算
- **推測で曜日を書くな。計算しろ。**
- 出力前に「○月○日 = ○曜日」を内部で確認してから文面に含める

### 2. メールの基本構造
```
{宛名}

お世話になっております。{差出人の所属・名前（必要な場合）}

{本文}

{署名（必要な場合）}
```

### 3. トーン
- 丁寧語（です・ます）が基本
- 過剰な敬語は避ける（「させていただきます」の連打NG）
- 簡潔に。1文が長くならないようにする
- 冒頭の挨拶は1行で済ませる

### 4. 日程調整メール
- 候補日は箇条書きで出す
- **日付（曜日）時間帯** の形式を統一する
- 例: `- 4月2日（木）終日可能`
- 相手が選びやすいように3候補以上出す（指定があればそれに従う）

### 5. 返信メール
- 相手のメール内容を踏まえて書く
- 遅れた場合は冒頭で一言謝る（長々と謝らない）
- 用件を先に、雑談は後（または省略）

### 6. 謝罪・訂正メール
- 何が間違っていたか明確にする
- 正しい情報を提示する
- 言い訳しない。短く済ませる

## 入力形式

ユーザーから以下の情報を受け取る（全部揃ってなくてもOK、文脈から判断）:

- **宛先**: メールアドレスまたは相手の名前
- **用件**: 何のメールか（日程調整、お礼、依頼、謝罪、返信など）
- **内容・条件**: 伝えたいこと、日時候補、制約条件など
- **元メール**: 返信の場合、元のメール文面

## 出力形式

```
---
宛先: {メールアドレス}
件名: {件名}
---

{メール本文}
```

- 件名は15文字以内を目安に簡潔に
- 本文はそのままコピペで送れる状態にする
- 署名は省略（ユーザーのメーラーで自動付与される前提）

## チェックリスト（出力前に必ず確認）

1. [ ] 日付と曜日が一致しているか **← 最重要**
2. [ ] 相手の名前・敬称は正しいか
3. [ ] 用件が明確に伝わるか
4. [ ] 不要な敬語や冗長表現がないか
5. [ ] コピペでそのまま送れる状態か
