# コンテンツ量産エンジン

ブログ記事を各SNS媒体に最適化したコンテンツに変換する。ユーザーの負担を最小化し、「おっけ」と言うだけで完結するフローを目指す。

---

## 起動トリガー

ユーザーが以下のいずれかを言ったら起動:
- `/content-engine`
- 「今日のコンテンツ」
- 「コンテンツ作って」
- 記事URLやIDを直接貼った場合

## プロファイル

`profiles/sassa.json` を読み込む。別アカウント対応時はプロファイルを追加するだけ。

---

## フロー

### Step 1: 素材選定

引数なしの場合、自動で素材を選ぶ:

1. `data/content-history.json` を読み、変換済み記事IDを取得
2. WordPress REST API (`https://sedorisassa.com/wp-json/wp/v2/posts?per_page=20&orderby=date&order=desc`) で最近の記事一覧を取得
3. 未変換の記事を優先度順にリストアップ
4. 最上位の記事を提案:
   ```
   📝 今日の素材候補:
   「記事タイトル」(ID: 1234)
   公開日: 2026-03-08
   → threads, note に最適（理由: ○○）
   ```

引数ありの場合（`/content-engine 記事ID:5340`）、指定記事を使用。

### Step 2: 記事取得・媒体判定

1. WP REST APIで記事本文を取得（HTMLタグ除去、構造化テキスト抽出）
2. 記事内容から最適な媒体を自動判定:
   - **ノウハウ系**（手順・方法）→ note + threads
   - **商品紹介系**（利益商品・レビュー）→ threads + X
   - **体験談系**（失敗談・成功談）→ threads + note
   - **ニュース系**（トレンド・速報）→ X + threads
3. YouTube台本は長文記事（3000字以上）のみ候補に

### Step 3: コンテンツ生成

各媒体ごとにコンテンツを生成する。以下のスキル・ルールを適用:

#### 共通ルール
- `sassa-writing.md` の文体ルールを厳守
- `anti-ai-writing.md` のAI臭消しルールを適用
- 元記事の情報を正確に使う。嘘は絶対に書かない
- 生成したら必ず自分でAI臭チェックを1回行い、引っかかったら自分で修正

#### Threads（500字以内）
- フック命。最初の1行で「おっ」と思わせる
- 連投スレッド形式も可（本文 → 補足 → CTA）
- ハッシュタグ3〜5個
- 「詳しくはブログで→」のCTA

#### note（2000〜4000字）
- SEOカニバリ対策: ブログと同一タイトル禁止。切り口を変える
- 見出し3〜5個、読みやすい構成
- 冒頭で「この記事でわかること」を箇条書き
- 末尾に「もっと詳しく知りたい方はブログへ」のCTA

#### X（140字以内）
- `x-strategy.md` のルールを厳守
- 短文インパクト。外部リンク絶対禁止
- セルフリプで補足・ブログ誘導はOK
- 商品系は金額インパクト（「○円→○円！！」）

#### YouTube台本（JSON形式）
- タイトル28字以内、数字+感情ワード
- スライド8〜12枚、narration + duration付き
- 台本のみ出力。動画デザインはCanva等で別途

### Step 5: 履歴記録

ユーザーが「おっけ」と言ったら、`data/content-history.json` に記録:
```json
{
  "articleId": 5340,
  "articleTitle": "記事タイトル",
  "date": "2026-03-10",
  "platforms": ["threads", "note", "x"],
  "status": "approved"
}
```

## 注意事項

- 生成前に確認を取るな。いきなり生成して結果を見せろ
- 「どの媒体にしますか？」と聞くな。自動判定して出せ
- 修正指示があれば該当部分だけ直せ。全部やり直すな
- content-engine CLI（`node src/index.js`）は使わない。Claude Code内で直接生成する
- API呼び出しは発生しない。Claude Codeのコンテキスト内で完結する
