# 駿河屋 買取リスト自動取得 - 引き継ぎ

## 目的
ひろさんのツール（入荷通知）に登録する利益商品リストを大量に作りたい。
現在100件 → 10,000件規模にしたい。

## アプローチ
駿河屋の買取価格リストから「買取価格が高い + 駿河屋に在庫がない + 他ショップなし」商品を抽出する。
→ 希少かつ高相場の商品リスト = オフモール等に入荷したら利益確定。

## 実装済みスクリプト

### 1. 買取リスト取得: `scripts/surugaya-kaitori-list.mjs`
- `node scripts/surugaya-kaitori-list.mjs [カテゴリID] [目標件数] [開始ページ] [ソート] [上限価格] [下限価格]`
- 品切れ+他ショップなしの商品を目標件数まで自動収集
- 途中保存（JSON+HTML）、再開対応、重複スキップ（knownIds）
- レート制限対策: 基本2秒待ち、50件ごとに5秒追加休憩
- 価格フィルタ対応（argv[6]=上限、argv[7]=下限）
- ソート対応（argv[5]）: price:descending / modificationTime:descending 等
- 実績: 155ページ・1881件チェックで100件収集（カテゴリ500、フィルタなし）

### 2. 入荷待ちリスト自動登録: `scripts/surugaya-nyuka-register.mjs`
- `node scripts/surugaya-nyuka-register.mjs [カテゴリID]`
- Puppeteerで実Chromeに接続して入荷待ちリストのボタンを自動クリック
- Chrome起動: リモートデバッグモード（専用プロファイル SurugayaBot、ポート9222）
- 手動ログイン必要（初回のみ。ログイン後Enterで続行）
- 登録済みID管理（kaitori_xxx_registered.json）で重複防止
- 実績: 100件中98件成功

### 3. ビューア: `scripts/kaitori-viewer.mjs`
- `node scripts/kaitori-viewer.mjs` → http://localhost:3002
- 累積リスト表示、UIからスクレイパー実行/停止、検索フィルタ

## 駿河屋の構造（確認済み）

### 買取リスト取得
- URL: `https://www.suruga-ya.jp/search_buy?category=カテゴリID&search_word=&page=ページ番号`
- ソート: `rankBy=` (デフォルト:関連順 / modificationTime:descending / release_date / purchase_price)
- **ascending/descending両方使える**（UIにはdescendingしかないがascendingもパラメータとして機能する）
- 20件/ページ
- `input.checkbox[data-product]` → `{"shinaban":"商品ID","kakaku":価格}`

### 在庫チェック（商品詳細ページ）
- 品切れ: `div.out-of-stock-text` あり
- 在庫あり: `form.add-cart` あり
- 他ショップ在庫: `a[href*="/product/other/"]` → 除外対象
- 買取価格: `span.price-buy`（在庫あり）/ `span.purchase-price`（品切れ）
- 販売価格: `span.text-price-detail`

### 入荷待ちリスト登録
- ボタン: `input.waitbtn`（品切れ商品ページ）
- fetchだとCloudflareに弾かれる → Puppeteer+実Chrome必須

## テスト結果まとめ

### テスト1: 買取価格の高い順（price:descending）100件
- 6ページ・102件チェックで100件収集（ヒット率98%）
- 問題: 160万〜15万円の超高額商品ばかり（ブリキ玩具、1/8ミニカー、超合金等）
- **売値5〜10万の現実的な仕入れ対象ではない**

### テスト2: 関連順（デフォルト）+ 価格フィルタ 1,000〜60,000円で100件
- **64ページ・30件チェックで3件しかヒットせず → 途中終了**
- 問題: 既知ID 2135件のスキップ + 価格帯フィルタで大半が対象外
- ページングが遅すぎて実用にならない

### ユーザーの判断
- **売値で狙える現実ラインは5〜10万**
- **→ 買取価格の上限は5〜6万が妥当**
- 下限は低くてOK

## 戦略方針（2026-03-23確定）

### ひろさんツール vs 自作の判断
- ひろさんツール: **スペース入りキーワードを認識しない致命的バグあり**
- ホビー系は型番ほぼなし → スペース区切り必須 → ひろさんツールはホビーに実質使えない
- **ホビー特化の入荷通知ツールを自作する方向で決定**
- コスト: Scrapfly無料枠（月1,000リクエスト）+ Railway月$5〜7程度

#### 釣り具即売れリスト取得結果（2026-03-23）
- 全209件取得（13ページ）→ `hiro_fishing_soldout.json`
- **型番フィールドあり: 63件**（ブランド付き。SHIMANO/DAIWA中心）
- **商品名に固有名詞あり（型番フィールドは空）: 53件**
  - 例: 「24ルビアス PC LT2500-H（00061216）」→ LT2500-Hが型番
  - 例: 「シーボーグ 100JL」→ 100JLが型番
  - **商品名の中から型番部分を抜き出す処理が未実装**
- **汎用名のみ: 93件**（「スピニングリール」「ヘラ竿」等。検索不可、捨て）
- 売り切れ商品でもオフモール商品ページは残っている（確認済み）
- ただしオフモールの商品名 = ひろさんツールの商品名（同じ情報源）。追加の型番情報はオフモールからも取れない

#### ひろさんツールの特性
- 単語1つしか認識しない（スペース区切り非対応）→ 型番ベースの登録が前提
- 即売れリスト: 新着入荷から10分後判定
- 日時売り切れリスト: 1日で売り切れたもの
- 一部ジャンルにはメルカリ差額表示あり（釣り具は非対応、比較0件ばかり）
- カテゴリ: 家電/玩具・ホビー/ゲーム/カード/パソコン/楽器/スマホ・タブレット/車・バイク/釣り/ゴルフ

#### 入荷通知登録の判断基準（ユーザー方針）
型番が判明した商品をメルカリで手動チェック:
1. **メルカリに履歴なし** → 登録（市場に出回ってない＝希少）
2. **履歴あり＋差額取れてる** → 登録
3. **履歴あり＋差額取れてない＋市場在庫枯れてる** → 登録

#### 次にやること
1. **型番ありの63件 + 商品名から型番抽出可能な53件 → メルカリ検索しやすいリスト（HTML）を生成**
   - メルカリ検索URLをワンクリックで開ける形
   - 商品名から型番部分を抽出するロジックが必要（「24ルビアス PC LT2500-H」→「LT2500-H」など）
2. 手動チェック後、登録すべき型番リストを確定
3. 確定した型番をひろさんツールの入荷通知に登録
