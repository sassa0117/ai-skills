# WordPress せどり長編記事 構成スキル

さっさのブログ（sedorisassa.com）における**有料記事の構成設計・ブロック設計・文体ルール**を定義するスキル。
記事を新規作成・リライトする際に参照し、構成案→構成承認→HTML生成の順で進めること。

参考記事:
- 4914（特撮おもちゃ 345K文字）
- 4522（フィギュア 143K文字）
- 4343（ぬいぐるみ 381K文字）

---

## 1. 記事構成フロー

**この順番を守ること。セクションの入れ替え・省略は必ずさっさに確認を取る。**

### 無料部分（全体の35-40%）

```
1. 導入（共感 + 結論提示）
   - 冒頭1-2文で「この記事は何か」を定義
   - 個人エピソード or 読者の誤解を否定して入る
   - 短い段落で結論を先出し

2. この記事を読むメリット（H2）
   - chevronリストで3-5項目
   - 読了後に得られるスキル・知識を具体的に列挙

3. 対象読者（H2）
   - 「向いている人」「向かない人」をリストで明示
   - ターゲットを絞ることで刺さりを強くする

4. なぜ今このジャンルが稼げるか（H2）
   - 市場構造の説明
   - 「供給」と「需要」のメカニズムを構造で見せる
   - H3で2-3個の理由に分解

5. ジャンルの特徴（H2: メリット / H2: デメリット・注意点）
   - メリットはchevronリストで列挙
   - デメリットは正直に書く（信頼構築）
   - 各項目をH3で展開

6. 初心者が最初に覚えるべきこと（H2 × 1-2セクション）
   - 具体的な作品 or ルールを1-2個だけ深掘り
   - 「法則」「概念」は教えるが「具体的な金額」は有料へ
   - 画像で見分け方を視覚的に示す

7. 実例チラ見せ（H2）
   - 無料で2-3商品の実売データを写真+テーブル付きで公開
   - 読者に「本当に利益が出る」ことを証明する
   - 解説は最小限（なぜ利益が出たかの深掘りは有料）

8. Q&A / よくある質問（H2）
   - sgb/faq or sgb/faq-itemで3-6問
   - 購入前の不安を解消する

9. 筆者プロフィール・実績（H2）
   - 実売データの画像で裏付け
   - 簡潔に

10. 有料部分で公開する内容 = お品書き（H2）
    - chevronリストで箇条書き
    - 具体的な数字を入れる（「実売事例40件以上」等）
    - さっさ吹き出しでクロージング（「ここから先は〜」）
```

### codocペイウォール

```html
<!-- wp:codoc/codoc-block {"showPrice":true,"price":3498,...} -->
<div data-id="codoc-tag" class="codoc-entries"></div>
<!-- /wp:codoc/codoc-block -->
```

### 有料部分（全体の60-65%）

```
11. すぐ使えるテクニック（H2）
    - 有料の最初は「即効性のある知識」で始める
    - 買った直後に「元が取れそう」と思わせる

12. 判断基準・フレームワーク（H2）
    - 扱う対象の選定基準
    - 仕入れ判断のフローチャート的な構造

13. ジャンル/作品別の深掘り（H2 × 複数セクション）
    - 各H2セクション内を統一構造にする:
      - H3: 仕入れ判断の考え方
      - H3: アイテムの判別方法
      - H3: 狙い目
      - H3: セット販売 or 販売戦略
      - H3: 失敗パターンと注意点
      - H3: 実売事例

14. 失敗例・やらかし集（H2）
    - 横断的な失敗パターン
    - 読者が同じ轍を踏まないための注意喚起

15. 実売事例集（H2）
    - H4で商品名 → 画像 → テーブル → 解説 → (任意)吹き出し
    - 40-60商品が目安
    - ジャンル/カテゴリ別にH3でグルーピング
```

## 2. ブロックエディターのルール

### H2（大章の区切り）

```html
<!-- wp:sgb/headings {"headingText":"{{見出し}}","headingStyle":"hh hh8","headingIconName":"","headingIconColor":"#333","headingBgColor1":"var(\\u002d\\u002dsgb-pastel-color)","headingBorderColor1":"var(\\u002d\\u002dsgb-main-color)"} -->
<h2 class="wp-block-sgb-headings sgb-heading"><span class="sgb-heading__inner hh hh8" style="background-color:var(--sgb-pastel-color);border-color:var(--sgb-main-color);font-size:1.2em"><span class="sgb-heading__text" style="color:#333">{{見出し}}</span></span></h2>
<!-- /wp:sgb/headings -->
```

### H3（節の区切り）

```html
<!-- wp:sgb/headings {"headingText":"{{見出し}}","headingStyle":"hh hh8","headingTag":"h3","headingIconName":"","headingIconColor":"#333","headingBgColor1":"whitesmoke","headingBorderColor1":"var(\\u002d\\u002dsgb-main-color)"} -->
<h3 class="wp-block-sgb-headings sgb-heading"><span class="sgb-heading__inner hh hh8" style="background-color:whitesmoke;border-color:var(--sgb-main-color);font-size:1.2em"><span class="sgb-heading__text" style="color:#333">{{見出し}}</span></span></h3>
<!-- /wp:sgb/headings -->
```

### H4（商品名見出し専用）

```html
<!-- wp:sgb/headings {"headingText":"{{商品名}}","headingStyle":"hh hh17","headingTag":"h4","headingIconName":"","headingTextColor":"","headingIconColor":"#333","headingBgColor1":"","headingBorderColor1":""} -->
<h4 class="wp-block-sgb-headings sgb-heading"><span class="sgb-heading__inner hh hh17" style="font-size:1.2em"><span class="sgb-heading__text">{{商品名}}</span></span></h4>
<!-- /wp:sgb/headings -->
```

### リストのスタイル使い分け

| スタイル | 用途 |
|---|---|
| `is-style-sango-list-chevron` | **基本**。要点の列挙、ポイントの並列提示 |
| `is-style-sango-list-main-color` | chevronの**子リスト（ネスト）**。詳細説明・補足 |
| `is-style-sango-list-circle` | 客観的事実の淡々とした列挙 |
| `is-style-sango-list-check` | sgb/block-simple内のチェックリスト限定 |
| `is-style-sango-list-yubi` | 具体的な商品例を指し示す用途（稀に使用） |

### 利益テーブル

wp-sedoriスキルのテンプレートを使用すること（css, scopedCSS, customControls含む完全版）。
blockIdは商品ごとにユニーク生成。行構成は以下の3パターン:

| パターン | 行 | 使い分け |
|---|---|---|
| A | 販売価格/仕入れ値/送料/利益 | 送料が明確な商品（基本） |
| B | 販売価格/仕入れ値/利益率/利益額 | 小物系（送料が小さい） |
| C | 買取価格/仕入れ値/利益 | 駿河屋買取パターン |

### さっさ吹き出し（sgb/say）

**配置ルール:**
- 無料部分: 各H2に0-1個。解説の補足・読者への励まし
- 有料部分: 商品解説の合間に。個人的な感想・雑談・豆知識
- **ペイウォール直前に1つ**（クロージング用、必須）
- 密度: 詰めすぎない。H2セクションに1つ程度

**トーン:**
- 無料部分の吹き出し: 丁寧語ベースだが親しみやすく
- 有料部分の吹き出し: カジュアル（「〜よね」「！！」「笑」OK）

### 質問キャラ吹き出し（任意）

読者の素朴な疑問を代弁する用途。使用頻度は低め（記事全体で0-2回）。

```html
<!-- wp:sgb/say {"avatarImageUrl":"https://sedorisassa.com/wp-content/uploads/2025/11/business_man3_1_question.png"} -->
```

### 関連記事ボックス（sgb/block-simple）

```html
<!-- wp:sgb/block-simple {"boxTitle":"\\u003cstrong\\u003e★関連記事\\u003c/strong\\u003e","boxTitleStyle":"sgb-box-simple\\u002d\\u002dtitle-overwrap","boxTitleColor":"var(\\u002d\\u002dwp\\u002d\\u002dpreset\\u002d\\u002dcolor\\u002d\\u002dsango-main)"} -->
```

内部に `▶ <a href="...">記事タイトル</a>` 形式でリンクを配置。

### お品書きボックス（ペイウォール直前）

```html
<!-- wp:sgb/block-simple {"boxTitle":"\\u003cstrong\\u003e【続きを読むと、こう変わります】\\u003c/strong\\u003e","boxTitleStyle":"sgb-box-simple\\u002d\\u002dtitle-overwrap","boxTitleColor":"var(\\u002d\\u002dwp\\u002d\\u002dpreset\\u002d\\u002dcolor\\u002d\\u002dsango-main)"} -->
```

内部にchevronリストで有料部分のベネフィットを列挙。

### 余白の取り方

**wp:spacerは使わない。** 以下のbrパターンで統一:

| 用途 | パターン |
|---|---|
| H3の前（小余白） | `<p><br><br></p>` |
| H2の前（大余白） | `<p><br><br><br></p>` |
| セクション末尾 | `<p><br><br></p>` |
| 最小の余白 | `<p></p>` |

### 強調の使い分け

| 手段 | 用途 |
|---|---|
| `<strong>` | **基本の強調**。キーワード、概念名、結論（全記事共通） |
| `<span class="green-highlight">` | **核心フレーズ**の視覚的マーカー。1セクションに0-2箇所。使いすぎ注意 |

## 3. 情報設計のルール

### 無料と有料の住み分け

| 無料で出すもの | 有料に回すもの |
|---|---|
| 「やり方」「法則」「概念」 | 「具体的な答え」「作品別の深掘り」 |
| 「何を見ればいいか」 | 「いくらで仕入れていくらで売れるか」 |
| 実例2-3点（証明用） | 実例40-60点（実践用） |
| パターン名 | パターンの具体的な適用方法 |
| 仕入れの考え方 | 仕入れの金額目安 |

**設計の本質:** 無料で「この人は構造で物を見ている」と思わせ、有料で「具体的な答え」を渡す。

### お品書きのルール

- 位置: ペイウォール直前のH2セクション
- 形式: chevronリストで箇条書き
- **具体的な数字を入れる**: 「実売事例40件以上」「作品別ガイド5本」等
- 曖昧にぼかさない。「何が得られるか」を明確にする

## 4. 文体ルール

### 基本トーン

- 一文は20-40文字が基本。100文字超の長文は避ける
- です・ます調
- 断言を恐れない（「〜です。」「〜します。」）
- 二人称は控えめ（「あなた」より「この記事を読むと〜」）

### 無料部分

- 丁寧語、客観的
- 教える人の立場
- 個人エピソードは冒頭の共感パートのみ
- green-highlightは無料部分に集中させてもOK

### 有料部分（フレームワーク解説）

- 丁寧語だがやや砕ける
- 「〜してください」「〜しておきましょう」と指示調が増える
- 判断基準を明確に提示

### 有料部分（実売事例）

- 一気にカジュアル
- 「〜ですね」「〜かなと思います」「〜って感じ」
- 一人称「僕」頻出
- 吹き出し内は最もカジュアル（「まじで」「！！」OK）

### 商品解説文の粒度

| 粒度 | 文字数 | 使い分け |
|---|---|---|
| 詳しい | 300字以上 | 初出の作品、特殊な仕入れ戦略 |
| 標準 | 100-200字 | 判断ポイントが明確な商品 |
| 簡潔 | 50字以下 | リピート事例、「見かけたらリサーチ」系 |

## 5. 商品カードの構成

有料部分の個別商品は以下の順番で構成する:

```
H4: 商品名
↓
wp:image（出品写真 1枚）
↓
wp:table（利益テーブル）
↓
wp:paragraph × 数個（解説文）
↓
（任意）sgb/say（さっさの感想）
↓
余白（<p><br><br></p>）
```

解説文に含める要素:
1. **リサーチのきっかけ**: なぜこの商品に反応したか
2. **利益が出た理由**: 供給停止、代替品不在、限定性等
3. **シリーズ全体への言及**: 同シリーズの他商品も狙えるか
4. **仕入れ判断の一般化**: この経験から他の商品にどう応用するか

書かないもの:
- 出品テクニック（タイトル・説明文の書き方）
- 正確な販売日
- 購入者の情報

---

## 6. 禁止事項

- Claude側で独自のノウハウや理論を追加しない（ソースはWP既存記事+note記事のみ）
- データの「某」「ある」ぼかし禁止（具体的な作品名・商品名・数字で記載）
- 情報商材的な煽りワード禁止（「門外不出」「解体新書」「根こそぎ奪う」等）
- 「登録方法」「発送方法」「梱包方法」など検索すれば出る情報は不要
- wp:spacer禁止（brパターンで余白管理）
- 構成の上書き禁止（必ず構成案を先に出してさっさの承認を取る）

---

## 7. 作業フロー

1. **ソース収集**: WP既存記事からデータ・ノウハウを抽出
2. **構成案を提示**: このスキルのフローに沿った見出し一覧を提示し、さっさの承認を得る
3. **承認後にHTML生成**: 構成案が承認されてから初めてHTMLを書く
4. **段階的に確認**: 一気に全文生成せず、セクション単位で確認を取りながら進める
5. **WP投稿**: 下書きとして保存し、プレビューで確認してもらう
