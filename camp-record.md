# 合宿記録・整理スキル

合宿（営業研修合宿）のボイスメモ・整理文・メモが渡されたら、以下を**すべて自動で**実行する。指示を待つな。

---

## トリガー

以下のいずれかが渡された時に自動発動:
- ボイスメモの文字起こし（原文）
- AI整理済みの文章
- 参加者のメモ（だい、りゅう等）
- 「合宿」「打ち合わせ」「MTG」に関する記録

---

## 自動実行フロー

### STEP 1: 既存ドキュメント確認
- `ai-skills/camp*-report.md`（該当する合宿のreportファイル）を読む
- メモリの合宿関連ファイル（`project_camp*-prep.md`）を確認
- 前回MTGとの差分を把握

### STEP 2: レポート更新（camp*-report.md）
新しいMTGの内容を既存レポートに**追記**（上書きしない）。
セクション番号を連番で振り、以下の構造で整理:

#### 必須セクション
1. **MTG概要**: 日時・参加者・テーマ
2. **ワーク詳細（確定版）**: 各ワークをテーブルで整理
3. **アシスタント役割分担（最終版）**: ワーク別マトリクス
4. **感情設計アーク**: フェーズ分けフロー
5. **個別戦略（参加者別）**: 課題・アプローチ・担当・前回比較
6. **運営詳細**: 時間/場所/食事/前日の動き
7. **アクション（更新）**: 完了→[x]、新規追加

#### ワーク詳細のテーブル形式
```markdown
| 項目 | 内容 |
|------|------|
| ルール | ... |
| 所要時間 | ... |
| 役割 | だい: ... / りゅう: ... |
| 意図 | ... |
| NG | ... |
```

### STEP 3: メモリ更新
- `project_camp*-prep.md` を最新情報で更新（上書き）
- MEMORY.mdのインデックスも更新
- 重複作成しない

### STEP 4: インフォグラフィック生成（2枚）
`/infographic` スキルのデザインシステムに従い、デスクトップに保存。

**1枚目: ワークフロー & 感情設計アーク**
- 左カラム: ワークの流れ（番号アイコン・名前・時間・キーポイント）
- 右カラム: 感情設計のフェーズ分け（バー + フェーズ区切り）
- 下部: まとめ帯

**2枚目: アシスタント役割分担マトリクス**
- 左カラム: アシスタントA（ワーク別行動）
- 中央: 連携の原則・個別指摘のクロス配置
- 右カラム: アシスタントB（ワーク別行動）
- 下部: まとめ帯

### STEP 5: PDF生成
- 印刷対応A4レイアウト（ページ区切りあり）
- Puppeteerで変換（`ai-skills/scripts/` にスクリプト配置）
- 出力先: デスクトップ
- 内容: ワーク詳細 + 役割分担 + 感情設計 + 個別戦略 + 運営詳細 + NGルール

### STEP 6: 完了報告
生成物の一覧をまとめて報告:
- レポートファイル更新箇所
- メモリ更新内容
- インフォグラフィック（デスクトップ、2枚）
- PDF（デスクトップ）

## インフォグラフィック CSS仕様

### 1枚目（ワークフロー）の構造
```
[タイトル帯] 合宿X ワークフロー & 感情設計アーク
[左: ワークリスト]              [右: 感情設計]
 ①番号アイコン+カード           PHASE 1: ...
 ↓                             バー██████
 ②番号アイコン+カード           PHASE 2: ...
 ↓                             バー██████████
 ...                           ...
[まとめ帯]
```

- ワーク番号: 色付き丸（bg-blue/green/teal/orange/purple/red/pink）
- 感情バー: 幅でテンション強度を表現（40%→55%→90%→75%→50%）
- 感情爆発ポイントはhighlight色（赤系）

### 2枚目（役割分担）の構造
```
[タイトル帯] 合宿X アシスタント役割分担マトリクス
[アシスタントA]  [連携の原則]  [アシスタントB]
 ヘッダー(色+名前+タグ)      ヘッダー(色+名前+タグ)
 ①行動               🔗     ①行動
 ②行動               連携カード  ②行動
 ...                 個別指摘   ...
[まとめ帯]
```

## PDF HTML仕様

### ヘッダー
- 濃い青背景（#1E40AF）、白文字
- タイトル・日時・参加者・対象者をメタ情報として表示

### 本文
- A4対応。font-size: 11px
- テーブル形式多用（ワーク詳細）
- アラートボックス（赤: NG/注意、青: 基本方針、黄: まとめ、緑: 連携パターン）
- フロー図（感情設計アーク）
- 2カラム（役割分担の左右比較）
- page-break-before: always で適切にページ区切り

---

## 注意事項

- ボイスメモの雑談・脱線は捨てない → 人間関係の力学や本音が含まれている
- だい等のメモは感情設計の核 → 必ず統合
- 前回合宿との比較・連続性を意識
- 合宿参加者の個人情報は外部に出さない（PDF共有は打ち合わせメンバー内のみ）
- 「松竹梅」は絶対に使うな
